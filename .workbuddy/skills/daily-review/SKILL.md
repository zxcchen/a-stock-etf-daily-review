# A股每日复盘报告生成器

> 使用通达信 MCP 数据，自动生成 A 股每日收盘报告：今日看板 + 复盘分析。

## 执行流程（严格按序）

### Step 1：确认日期和交易状态

1. 获取当前日期 `TODAY`（YYYYMMDD 格式）
2. 调用 `tdx_kline` 查询上证指数 000001 最近 1 根日 K 线，确认是交易日且有最新收盘数据
3. 如果当日非交易日或无数据，告知用户并结束

### Step 2：拉取核心数据（并行请求全部）

一次性并行请求以下所有数据：

**指数行情：**
- `tdx_quotes`：codes=["000001","399001","399006","000300","000688"]，获取上证/深证/创业板/沪深300/科创50

**四大 ETF 行情 + 近 5 日 K 线：**
- `tdx_quotes`：codes=["510300","159915","588000","512480"]
- `tdx_kline`：对上述 4 个 code 各调一次，period="day"，count=6，获取近 5 日 + 今日数据

**市场情绪：**
- `tdx_quotes`：codes=["880005","880006","880008"] 获取涨跌家数等市场统计
- `tdx_api_data`：entry="TdxSharePCCW.tdxf10_gg_rdtc"，code="沪深京A股"，fixedTag="zttzbkz"

**行业 ETF 异动（6 只）：**
- `tdx_quotes`：codes=["512400","515220","159611","515880","512010","512880"]

**资金流向：**
- `tdx_quotes`：codes 对应板块

**融资融券：**
- `tdx_api_data`：查询两融数据

**跨市场数据：**
- `tdx_quotes`：查询美元指数、黄金、原油、恒生等

**新闻资讯：**
- `wenda_news_query`：keyword="A股"，date_range="today"，limit=5

**政策消息：**
- `wenda_news_query`：keyword="政策"，date_range="last_3_days"，limit=5

### Step 4：生成模版2 — 复盘分析

在模版1 数据基础上生成复盘分析，保存到 `analysis/YYYYMMDD.md`，同时更新 `latest_analysis.md`。

复盘分析需要包含以下核心内容：

1. **指数趋势**：上证/科创50/创业板的当前状态、关键支撑压力位、操作判断
2. **ETF趋势**：4只ETF的状态、关键位置、操盘建议
3. **ETF放量倍数（B条件检测）**：检测结果
4. **仓位状态建议**：根据3万元建仓计划给出每只ETF的仓位建议
5. **其他方向重点**：市场风格切换、主线分析、核心判断

格式遵循用户提供的示例风格——口语化、有判断力、结合模型和仓位给出具体操作建议。

**关键分析逻辑：**
- 基于"ETF异动择时模型 + 趋势跟踪"
- 放量倍数 ≥ 1.5x 为B条件满足
- 两市成交 > 2万亿为C条件满足
- 下跌家数 > 3500为A条件满足
- 星级评定：0星（无B条件）→ 不做拐点判断；满足条件越多星级越高
- 仓位管理基于3万元总资金计划

## 数据获取细则

### 指数数据字段
- tdx_quotes 返回：name, last, pct_chg, amount（成交额）, volume
- 上证: 000001, 深证: 399001, 创业板: 399006, 沪深300: 000300, 科创50: 000688

### ETF K线数据
- 用 tdx_kline 获取近6根日K线（含今日），提取 open/close/high/low/amount
- 计算近5日均值（不含今日）作为放量基准
- 换手率从 tdx_quotes 获取

### 涨跌家数
- 从 tdx_quotes 查询 880005（上涨家数指数）、880006（下跌家数指数）

### 颜色规则
- 涨：红色 `<span style="color:red">+XX.XX%</span>`
- 跌：绿色 `<span style="color:green">-XX.XX%</span>`
- 这是中国股市惯例

### 涨停/跌停数据
- 通过 tdx_api_data 获取涨停板数据
- 若无接口返回，标注"数据待补录"

### 资金流向
- 通过 tdx_api_data 获取板块资金流向
- entry="TdxSharePCCW.skef10_bk_cpbd_jczl" 获取板块基础资料

## 注意事项

1. **交易日检测**：生成前必须确认当日为交易日且有有效收盘数据
2. **数据缺失处理**：个别数据无法获取时标注"数据待补录"，不阻塞报告生成
3. **模版1 是纯数据看板**：只展示数据和分析，不给投资建议
4. **模版2 是策略分析**：结合模型、仓位、趋势给出判断
5. **GitHub 推送**：每次生成后必须推送
6. **报告时间标注**：记录报告生成的实际时间
